Treatment Recommender Widget for WordPress

This page demonstrates how to embed the treatment recommender widget in WordPress.

WordPress File Upload Restriction

WordPress blocks direct uploads of JavaScript (.js) files for security reasons. Below are two solutions to this limitation.

Solution 1: Use a Plugin to Allow JS Uploads (Recommended)

Install the "WP Extra File Types" plugin or similar that allows JavaScript files to be uploaded to the media library:

  1. Go to Plugins → Add New and search for "WP Extra File Types"
  2. Install and activate the plugin
  3. Go to Settings → WP Extra File Types
  4. Check the box for JavaScript (.js) files
  5. Save changes
  6. Now you can upload the .js and .css files to your media library

Solution 2: Host the Files Externally

Host your JS and CSS files on an external service that allows JavaScript files:

  1. Upload the files to a service like GitHub Pages, Amazon S3, or your own web server
  2. Make sure the files are publicly accessible
  3. Use the direct URLs to these files in your WordPress integration

Solution 3: Add Files via FTP

Upload files directly to your WordPress site using FTP:

  1. Connect to your WordPress site using an FTP client (like FileZilla)
  2. Navigate to wp-content/uploads/custom-widget/ (create this folder if it doesn't exist)
  3. Upload the treatment-recommender.umd.js and style.css files
  4. Use the direct URLs to these files in your WordPress integration

WordPress Integration Instructions

After making the JS file accessible using one of the methods above, add the following code to your WordPress page using a Code Block or HTML Block:

<!-- React Dependencies --> <script src="https://unpkg.com/react@18/umd/react.production.min.js" crossorigin></script> <script src="https://unpkg.com/react-dom@18/umd/react-dom.production.min.js" crossorigin></script> <!-- Widget Files - Update these URLs with your actual file locations --> <script src="https://YOUR-DOMAIN.com/wp-content/uploads/custom-widget/treatment-recommender.umd.js"></script> <link rel="stylesheet" href="https://YOUR-DOMAIN.com/wp-content/uploads/custom-widget/style.css"> <!-- Widget Container --> <div id="treatment-recommender-container" style="width: 100%; min-height: 800px;"></div> <script> // Configure EmailJS window.TreatmentRecommender.configureEmailService({ serviceId: "your_emailjs_service_id", templateId: "your_emailjs_template_id", userId: "your_emailjs_user_id" }); // Configure Widget Settings window.TreatmentRecommender.configureSiteSettings({ title: "YOUR BRAND NAME", subtitle: "Your custom subtitle here", copyrightName: "Your Brand", layout: { showHeader: true, showFooter: true } }); // Render the widget document.addEventListener('DOMContentLoaded', function() { var container = document.getElementById('treatment-recommender-container'); if (window.ReactDOM.createRoot) { ReactDOM.createRoot(container).render( React.createElement(TreatmentRecommender.default) ); } else { ReactDOM.render( React.createElement(TreatmentRecommender.default), container ); } }); </script>

WordPress with Elementor Specific Instructions

If you're using Elementor Page Builder:

  1. Edit your page with Elementor
  2. Add an "HTML" widget to your page
  3. Paste the code above into the HTML widget
  4. Save and update the page

WordPress with Divi Builder Specific Instructions

If you're using the Divi Builder:

  1. Create a new page or edit an existing one
  2. Add a "Code Module" element
  3. Paste the code above into the module
  4. Important: Make sure "Disable on-page builder" is checked in the module settings
  5. Save and publish the page

Troubleshooting

If the widget doesn't appear:

  1. Check your browser console for errors (F12 or right-click → Inspect → Console)
  2. Verify that your JavaScript and CSS file URLs are correct
  3. Make sure the files are accessible by directly visiting their URLs in your browser
  4. Confirm your FTP upload or plugin configuration was successful
  5. If using a page builder, make sure it's not adding extra HTML that interferes with the script
